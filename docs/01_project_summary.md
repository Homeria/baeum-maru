# 배움마루 프로젝트 요지

## 1. 프로젝트명

- 한글명: **배움마루**
- 영문명: **Baeum-Maru**
- 저장소명 후보: `baeum-maru`
- 실행 파일명 후보: `BaeumMaru.exe`

## 2. 프로젝트 정의

배움마루는 복지관, 노인문화센터, 문화센터, 평생교육기관 등에서 사용하는 **수강신청·추첨·대기자·출석부 관리용 로컬 호스팅 런처**이다.

기존 엑셀 기반 수작업을 대체하거나 보조하여, 접수 기간 동안 여러 직원이 동시에 수강신청을 입력하고, 마감 후 자동 추첨 및 결과 정리를 수행할 수 있도록 한다.

## 3. 문제 배경

기존 수강신청 업무는 다음과 같은 방식으로 진행되는 경우가 많다.

- 엑셀 파일로 회원명단 관리
- 종이 신청서 또는 수기 접수
- 강좌별 신청 여부를 사람이 직접 표시
- 정원 초과 여부 수동 확인
- 중복 신청 수동 확인
- 반장/우선순위/제한 규칙 수동 반영
- 추첨 후 결과와 대기자 명단 수동 정리
- 강좌별 출석부 엑셀 수동 생성

이 방식은 강좌 수와 신청자 수가 늘어날수록 실수 가능성이 커지고, 여러 명이 동시에 작업하기 어렵다.

## 4. 프로젝트 목표

### 4.1 핵심 목표

- 수강신청 업무를 엑셀보다 안전하고 편하게 처리한다.
- 한 대의 노트북을 임시 서버처럼 사용해 같은 내부망의 접수용 노트북들이 접속할 수 있게 한다.
- 담당 직원이 별도 개발환경 없이 `exe`만 실행하여 사용할 수 있게 한다.
- 수강신청, 추첨, 대기자, 출석부, 엑셀 출력까지 한 사이클을 완성한다.
- 실데이터를 외부 클라우드에 저장하지 않는다.

### 4.2 장기 목표

- 다른 복지관/문화센터도 사용할 수 있는 오픈소스 프로젝트로 공개한다.
- 기관별 규칙을 설정할 수 있게 한다.
- 상시 서버 모드로 확장할 수 있게 한다.
- Docker 기반 서버 배포를 지원할 수 있게 한다.
- 추후 설치 지원, 데이터 이전, 커스터마이징, 유지보수 모델로 확장할 수 있다.

## 5. 프로젝트 성격

배움마루는 다음 중 하나에만 해당하지 않는다.

- 단순 데스크탑 앱
- 일반 클라우드 SaaS
- 단일 사용자 엑셀 대체 프로그램
- 대규모 LMS

정확한 성격은 다음과 같다.

> **네이티브 런처가 내장 웹서버와 SQLite DB를 제어하고, 같은 내부망의 여러 접수 PC가 브라우저로 접속하는 포터블 로컬 호스팅 업무 시스템**

## 6. 주요 사용 시나리오

### 6.1 접수 기간

1. 담당 직원이 호스팅 노트북에서 `BaeumMaru.exe`를 실행한다.
2. 네이티브 컨트롤 패널이 열린다.
3. IP, 포트, DB 상태, 최근 백업 시간을 확인한다.
4. `[서버 시작]`을 누른다.
5. 컨트롤 패널에 표시된 주소를 접수용 노트북 2~4대에서 연다.
6. 접수 직원들이 브라우저에서 회원을 검색하고 수강신청을 입력한다.
7. 접수 종료 후 담당자가 서버를 중지하거나 백업 후 종료한다.

### 6.2 추첨 기간

1. 담당자가 접수 마감 상태로 전환한다.
2. 시스템이 강좌별 신청자 수, 정원 초과 여부, 제한 위반 여부를 확인한다.
3. 추첨 대상 강좌를 확인한다.
4. 자동 추첨을 실행한다.
5. 당첨자, 대기자, 탈락자 결과를 저장한다.
6. 강좌별 결과표와 대기자 명단을 엑셀로 출력한다.

### 6.3 수업 운영

1. 강좌별 출석부를 생성한다.
2. 강좌별 수강자 명단을 출력한다.
3. 필요 시 수강 이력을 조회한다.

## 7. 이름에 대한 기준

`배움마루`는 다음 이유로 선택한다.

- `강좌`나 `수강`보다 부드럽고 확장성이 있다.
- 한국 복지관/문화센터 분위기와 잘 맞는다.
- `마루`는 공간, 중심, 플랫폼 느낌을 준다.
- 수강신청뿐 아니라 출석, 통계, 운영관리까지 확장해도 어색하지 않다.

상표나 이름 충돌 가능성에 대비해 내부 코드 구조는 이름 변경이 쉽도록 작성한다.

- 표시명은 설정값 또는 상수로 관리한다.
- DB 테이블, API 경로, 핵심 패키지명에는 `baeum_maru`를 과하게 박지 않는다.
- 내부 패키지는 `app`, `launcher`, `server`, `course`, `registration` 등 도메인 중심으로 구성한다.
