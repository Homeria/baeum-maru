# MVP와 프로토타입 범위

## 용어

- Go 기능 검증 구현: `go-prototype-baseline-2026-07` 태그에 보존된 과거 참고 구현
- Python 프로토타입: 목표 사용자와 Windows 환경에서 실제 업무, UI, 보안, 배포를 검증할 활성 구현

Go 구현의 기능 수를 Python 구현의 완료로 간주하지 않는다. Python에서 자동 테스트와 운영 흐름으로 다시 검증된 기능만 완료로 표시한다.

## Python 기반 필수 범위

- 회원, 회차, 강좌, 복수 시간표, 공간 기준 데이터
- 회원 정보와 여러 강좌 신청을 한 transaction으로 저장하는 접수
- 중복, 최대 신청 수, 시간 충돌, 접수 상태 규칙
- 신청 조회, 취소, 추첨, 대기 순번, 확정과 자동 승격
- 출석 회차와 출석 기록
- Excel 가져오기/내보내기
- SQLite 백업, 복구 예약, 감사 로그
- 접속 코드 인증과 직원/임시 직원/조회 전용 권한
- React 업무 화면과 WebSocket 기반 다중 사용자 갱신
- pywebview 독립 런처와 내부망 업무 서버 프로세스 제어
- HTTPS, CSRF, 로그인 실패 제한
- Python 미설치 Windows PC에서 실행되는 PyInstaller `onedir` 포터블 ZIP

## 프로토타입 완료 기준

다음 흐름을 담당자가 사용자 가이드만 보고 수행할 수 있어야 한다.

1. 포터블 폴더의 실행 파일을 열고 pywebview 독립 런처를 확인한다.
2. 네트워크와 기관 설정을 확인하고 업무 서버를 시작한다.
3. 접속 코드를 발급하고 직원 2~5명이 브라우저에서 로그인한다.
4. 회원을 검색하거나 등록하고 여러 강좌를 한 번에 신청한다.
5. 다른 직원 화면에서 신청 현황과 정원이 안전하게 갱신된다.
6. 접수를 마감하고 추첨, 확정, 대기자 승격, 출석, Excel 출력을 수행한다.
7. 백업을 만들고 복구 절차를 확인한 뒤 서버를 안전하게 종료한다.
8. 일반 사무용 Windows 노트북에서 Python이나 Node.js 설치 없이 같은 흐름을 재현한다.

## 프로토타입 이후

- Docker 기반 중앙 FastAPI 서버
- PostgreSQL adapter
- Google Drive/메일 백업과 현황 알림
- 행사·공간 디스플레이
- 설치 관리자 또는 자동 업데이트

이 항목은 프로토타입 필수 범위가 아니며 핵심 운영 리허설 이후 별도 로드맵으로 다룬다.
