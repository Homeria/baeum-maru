# MVP와 프로토타입 범위

## 용어

- 기능 MVP: 회원부터 백업/복구까지 한 업무 사이클의 도메인 기능이 동작하는 상태
- 프로토타입: 목표 사용자와 목표 환경에서 제품 구조, UI, 보안, 운영 흐름을 검증할 수 있는 상태

기능 MVP는 대부분 구현됐다. 이제 프로젝트의 목표는 빠른 데모가 아니라 프로토타입 완성이다.

## 기능 MVP 완료 범위

- 회원, 강좌, 신청, 제한 규칙, 추첨, 대기자, 출석, Excel, 백업/복구
- SQLite 기반 업무 데이터, 접속 코드 인증, 역할 권한, 감사 로그
- Fyne 기반 호스트 운영 화면과 포터블 패키지 기반

## 프로토타입 필수 범위

- React/Vite/TypeScript 기반 브라우저 업무 화면
- Huma v2/OpenAPI 기반의 버전 관리 REST API와 생성된 TypeScript client
- 회원 정보와 다중 강좌 선택을 한 접수 흐름으로 처리
- 시간표와 공간 기준 데이터를 반영한 강좌 운영 UI
- API와 SSE 기반의 다중 사용자 최신 상태 반영
- Wails v2 기반 Windows 호스트 런처
- WebView2 부재 시 콘솔 서버 fallback으로 브라우저 업무를 지속할 수 있는 포터블 패키지
- HTTPS, CSRF, 로그인 실패 제한을 포함한 내부망 보안 기준
- Windows 사무용 노트북과 2~5개 브라우저에서 실제 업무 시나리오 검증
- 백업/복구와 Excel 결과를 포함한 한 회차 운영 리허설

## 프로토타입 완료 기준

다음 흐름을 담당자가 사용자 가이드만 보고 수행할 수 있으면 프로토타입이 완료된 것으로 본다.

1. 런처에서 서버를 시작하고 내부망 주소와 접속 코드를 발급한다.
2. 직원 여러 명이 브라우저에서 로그인한다.
3. 회원을 검색하거나 등록하고 여러 강좌를 선택해 신청한다.
4. 다른 직원 화면에서 신청 현황과 정원이 최신 상태로 갱신된다.
5. 접수를 마감하고 추첨, 확정, Excel 출력을 수행한다.
6. 백업을 만들고 복구 절차를 검증한 뒤 서버를 안전하게 종료한다.
