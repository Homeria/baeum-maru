# 배움마루 품질 체크리스트

## 1. 목적

이 문서는 배움마루를 개발할 때 매 단계 확인해야 할 품질 기준을 정리한다.

완벽한 엔터프라이즈 시스템을 목표로 하지 않는다. 대신 작은 포터블 업무 도구가 지켜야 할 기본 안정성, 데이터 보호, 복구 가능성을 우선한다.

## 2. 공통 기준

- 앱은 명령어 없이 실행 가능해야 한다.
- 오류 메시지는 사용자용 안내와 개발자 로그를 분리한다.
- DB 변경은 마이그레이션으로 관리한다.
- 런타임 데이터는 Git에 포함하지 않는다.
- 더미 데이터와 실데이터를 섞지 않는다.
- 기능 추가 후 최소 수동 시나리오를 실행해 본다.

## 3. 데이터 보호

- 주민등록번호 전체 저장은 지원하지 않는다.
- 전화번호와 생년월일은 필요한 화면에만 표시한다.
- 로그에 개인정보를 과도하게 남기지 않는다.
- 엑셀 출력 파일은 개인정보 파일임을 안내한다.
- 백업 파일 관리 주의 문구를 제공한다.
- 스크린샷을 공개할 때 개인정보를 제거한다.

## 4. DB와 트랜잭션

- SQLite는 WAL 모드를 사용한다.
- foreign key를 활성화한다.
- busy timeout을 설정한다.
- 신청 저장은 트랜잭션으로 처리한다.
- 추첨 실행은 하나의 일관된 작업 단위로 처리한다.
- 추첨 중에는 관련 신청 수정을 막는다.
- 복구 전 현재 DB를 자동 백업한다.

## 5. 서버와 네트워크

- 서버 중복 시작을 막는다.
- 서버 중지 시 graceful shutdown을 수행한다.
- 포트 사용 중이면 사용자 친화 메시지를 표시한다.
- `127.0.0.1`과 내부망 IP의 차이를 안내한다.
- 외부 포트포워딩은 기본 지원하지 않는다.
- 방화벽 안내 문구를 제공한다.

## 6. 웹 UI

- 접수 화면은 키보드와 마우스 모두 빠르게 사용할 수 있어야 한다.
- 검색 결과와 신청 목록은 부분 갱신한다.
- 저장 성공/실패 메시지가 명확해야 한다.
- 버튼을 누른 뒤 중복 제출을 막는다.
- 표는 정렬과 필터가 가능해야 한다.
- 긴 이름, 긴 강좌명, 작은 화면에서도 레이아웃이 깨지지 않아야 한다.

## 7. 추첨

- 추첨 전 대상 강좌와 신청 수를 보여준다.
- 추첨 seed를 저장한다.
- 중복 추첨 시 경고한다.
- 결과를 저장한 뒤 신청 상태와 결과 테이블의 일관성을 확인한다.
- 결과 엑셀 출력과 화면 조회 결과가 일치해야 한다.

## 8. 엑셀

- 표준 양식 샘플을 제공한다.
- 필수 컬럼 누락 시 명확히 안내한다.
- 잘못된 행은 행 번호와 이유를 보여준다.
- 가져오기 원본 파일은 영구 저장하지 않는다.
- 내보낸 파일명은 날짜와 종류를 포함한다.
- 엑셀 열 너비와 헤더는 인쇄 친화적으로 만든다.

## 9. 백업/복구

- 수동 백업이 가능해야 한다.
- 백업 후 종료가 가능해야 한다.
- 최근 백업 시간이 표시되어야 한다.
- 백업 파일명이 중복되지 않아야 한다.
- 복구 전 현재 DB 백업을 만든다.
- 복구 후 앱을 재실행하지 않아도 상태를 확인할 수 있어야 한다.

## 10. 테스트 기준

### MVP-0

- 서버 실행
- DB 생성
- 회원 등록/검색
- 강좌 등록
- 신청 등록
- 신청 목록 조회

### MVP-1

- 중복 신청 차단
- 시간 중복 신청 차단
- 최대 신청 수 제한
- 접수 마감 후 수정 차단

### MVP-2

- 정원 이하 자동 확정
- 정원 초과 추첨
- 대기자 순번 생성
- 결과 엑셀 출력

### MVP-3

- 런처에서 서버 시작/중지
- 접속 주소 복사
- 백업 후 종료
- 백업 복구
- portable ZIP 실행

## 11. Git 기준

- `main`은 가능하면 실행 가능한 상태로 유지한다.
- 문서만 변경한 커밋은 `docs:` prefix를 사용한다.
- 기능 추가는 작은 단위로 커밋한다.
- 실데이터, DB 파일, 백업 파일, 엑셀 출력 파일은 커밋하지 않는다.
- 큰 기능은 별도 브랜치에서 작업한다.

## 12. 오픈소스 공개 전

- LICENSE를 선택한다.
- README에 설치/실행 방법을 적는다.
- 개인정보 주의 문구를 추가한다.
- 더미 데이터만 포함한다.
- 스크린샷 개인정보를 제거한다.
- 상표와 이름 충돌을 간단히 확인한다.
- config 예시에서 비밀번호를 제거한다.
