# 배움마루 요구사항 정의서

## 1. 대상 환경

### 1.1 기본 사용 환경

- OS: Windows 사무용 노트북
- 실행 방식: 포터블 ZIP 압축 해제 후 `BaeumMaru.exe` 실행
- 서버 역할: 실행한 노트북이 내부망 호스트 역할
- 접속 방식: 같은 내부망의 다른 노트북들이 브라우저로 접속
- 동시 접속 규모: 호스팅 노트북 포함 최대 5대
- 외부 공개: 기본적으로 하지 않음
- DB: 로컬 SQLite 파일

### 1.2 추후 확장 환경

- 상시 서버 PC
- Ubuntu Server
- Docker 배포
- Linux binary 배포
- SQLite 또는 PostgreSQL 선택 가능성

## 3. MVP 기능 요구사항

## 3.1 런처 컨트롤 패널

네이티브 프로그램 창으로 제공한다. 브라우저 기반 대시보드가 아니다.

필수 표시 정보:

- 프로그램명
- 서버 상태: 중지됨 / 실행 중 / 오류
- 호스트 IP
- 포트 번호
- 접속 주소
- DB 상태
- DB 파일 위치
- 최근 백업 시간
- 접속 중인 클라이언트 수
- 로그 요약 또는 최근 오류

필수 버튼:

- 서버 시작
- 서버 중지
- 관리 화면 열기
- 접수 화면 열기
- 접속 주소 복사
- 백업하기
- 백업 폴더 열기
- 로그 보기
- 설정
- 백업 후 종료

## 3.2 내부망 호스팅

- `[서버 시작]` 버튼을 누르면 지정 IP/포트에서 웹서버를 시작한다.
- 기본 포트는 `18080`으로 한다.
- 같은 내부망 PC에서 `http://호스트IP:포트`로 접속 가능해야 한다.
- Windows 방화벽 안내 문구를 제공한다.
- 포트 사용 중이면 오류를 표시하거나 다른 포트를 제안한다.
- 서버 실행 중에는 중복 시작을 막는다.
- 서버 중지 시 graceful shutdown을 수행한다.

- 생년월일
- 연락처

필수 기능:

- 회원 검색
- 회원 수정

- 민감정보 저장은 최소화한다.

필수 기능:

## 3.6 신청 제한 규칙

MVP에서는 최소한 다음 규칙을 지원한다.

- 1인 최대 신청 강좌 수
- 동일 시간대 중복 신청 제한
- 동일 강좌 중복 신청 제한
- 접수 마감 후 신청 수정 제한

추후 확장 가능한 규칙:

- 1인 최대 당첨 강좌 수
- 강좌 분야별 중복 제한
- 반장/봉사자/우선순위 대상자 우선
- 1순위 당첨 시 2순위 제외
- 미달 강좌 자동 당첨
- 기관별 커스텀 추첨 규칙

## 3.7 추첨 관리

필수 기능:

- 강좌별 신청자 수 확인
- 정원 초과 강좌 표시
- 추첨 대상 강좌 선택
- 자동 추첨 실행
- 당첨자 저장
- 대기자 순번 생성
- 탈락자 저장
- 추첨 결과 조회
- 추첨 결과 엑셀 출력
- 추첨 실행 로그 기록

추첨 관련 주의:

- 추첨 실행 전 백업 권장
- 추첨 실행 후 결과 수정은 관리자 권한으로 제한
- 같은 강좌에 대해 중복 추첨 시 경고 표시
- 추첨 결과는 재현 가능성을 위해 로그 또는 seed 기록을 고려한다.

필수 기능:

- 당첨 취소 시 대기자 승격 기능은 추후 고려

## 3.9 출석부 및 출력

필수 기능:

- 강좌별 수강자 명단 생성
- 강좌별 출석부 생성
- 엑셀 파일로 내보내기
- 인쇄 친화적인 출력 형식 제공

## 3.10 엑셀 입출력

필수 기능:

- 회원 명단 가져오기
- 강좌 목록 가져오기
- 신청 결과 내보내기
- 강좌별 신청자 명단 내보내기
- 당첨자/대기자 명단 내보내기
- 출석부 내보내기

정책:

- 업로드한 엑셀 원본은 파싱 후 영구 저장하지 않는다.
- 필요한 경우 `imports/` 폴더에 임시 저장 후 삭제한다.
- 엑셀 양식이 다를 수 있으므로 추후 매핑 기능을 고려한다.

## 3.11 백업/복구

필수 기능:

- 수동 백업
- 종료 전 백업
- 최근 백업 시간 표시
- 백업 폴더 열기
- 백업 파일에서 복구

백업 파일명 예시:

```text
center_2026-07-01_0930.db
center_2026-07-01_1730.db
```

주의:

- 실행 중 단순 파일 복사보다 SQLite backup API 또는 안전한 방식 사용
- 추후 백업 암호화 고려
- 복구 전 현재 DB 백업 생성 권장

## 3.12 로그

필수 로그:

- 프로그램 실행
- 서버 시작
- 서버 중지
- 로그인 성공/실패
- 회원 등록/수정
- 강좌 등록/수정
- 신청 등록/수정/취소
- 추첨 실행
- 백업 성공/실패
- 오류 발생

주의:

- 로그에 전화번호, 생년월일 등 개인정보를 과도하게 남기지 않는다.
- 가능하면 내부 ID 중심으로 기록한다.

## 4. 비기능 요구사항

## 4.1 사용 편의성

- 담당자가 명령어를 입력하지 않아야 한다.
- 별도 프로그램 설치 없이 실행되어야 한다.
- 컨트롤 패널에서 접속 주소를 쉽게 복사할 수 있어야 한다.
- 오류 메시지는 개발자용 로그와 사용자용 안내를 분리한다.
- 접수 화면은 빠르게 입력할 수 있어야 한다.

## 4.2 성능

대상 규모:

- 동시 접속 최대 5대
- 회원 수 수천~수만 명 가능
- 강좌 수 수십~수백 개
- 신청 건수 수천~수만 건 가능

성능 목표:

- 회원 검색은 체감상 빠르게 동작
- 신청 저장은 즉시 처리
- 엑셀 출력은 수초 내 처리
- 추첨은 수초~수십 초 내 처리 가능

## 4.3 안정성

- 서버 중지 시 graceful shutdown
- DB 트랜잭션 사용
- 추첨 중 신청 수정 잠금
- 백업 전후 상태 확인
- WAL 모드 사용
- busy timeout 설정

## 4.4 보안

- 외부 인터넷 공개 기본 비활성화
- 내부망 접속만 전제
- 관리자 로그인 적용
- 직원별 계정은 추후 적용 가능
- DB 파일 접근 권한 주의
- 백업 파일 관리 주의
- 미디어 업로드 금지
- 개인정보 로그 최소화
