# 배움마루 요구사항

## 대상 환경

- 호스트: 일반적인 Windows 10/11 복지관 사무용 노트북
- 이용자: 호스트를 포함해 최대 5명 정도의 직원/자원봉사자
- 네트워크: 강의실 Wi-Fi, 기관 내부망, 핫스팟을 포함한 신뢰 수준이 낮을 수 있는 내부망
- 저장소: 호스트 노트북의 SQLite 파일
- 배포: 포터블 ZIP 또는 설치 패키지. 직원용 별도 앱 설치는 하지 않는다.

## 사용자

| 사용자 | 주된 업무 |
|---|---|
| 호스트 담당자 | 서버 시작/중지, 접속 주소, 접속 코드, 설정, 백업/복구, 로그 |
| 직원 | 회원/강좌 관리, 접수, 신청 현황, 추첨, 엑셀 출력 |
| 임시 직원 | 접수와 필요한 조회 |
| 조회 전용 | 신청 현황 등 읽기 전용 조회 |

## 핵심 업무 요구사항

### 초기 운영 설정

- 건물, 층, 공간, 공간 역할을 기준 데이터로 등록하고 재사용한다.
- 회차, 강좌 분야, 과목, 강사, 시간대, 강좌 개설을 분리해 운영한다.
- 강좌 개설은 여러 요일/시간대와 공간을 조합할 수 있다.

### 회원과 접수

- 회원번호, 이름, 연락처, 성별, 선택 강좌를 입력한다.
- 회원 검색 결과를 선택하거나, 같은 화면에서 새 회원을 등록한다.
- 한 접수 작업에서 최대 신청 가능 수만큼 강좌를 선택하고 한 번에 저장한다.
- 중복 강좌, 시간 충돌, 접수 마감, 최대 신청 수를 저장 시점에 검사한다.
- 다른 사용자의 저장으로 정원이나 신청 목록이 바뀌면 최신 상태를 반영한다.

### 운영 결과

- 강좌별 신청 현황, 선정/대기/탈락/확정 상태를 조회한다.
- seed가 기록되는 추첨과 대기자 승격을 지원한다.
- 회원, 강좌, 신청, 추첨 결과를 Excel로 출력한다.
- 백업 생성, 목록 조회, 복구 예약과 감사 로그 조회를 지원한다.

## 호스트 런처 요구사항

- 실행 파일을 열면 기본 브라우저가 아니라 pywebview 기반 독립 런처 창이 열린다.
- 런처가 열린 직후 직원용 업무 서버는 자동 시작하지 않고 `중지` 상태를 유지한다.
- 대시보드, 서버 제어, 내부망 접속 주소, 접속 코드, 로그, 공간 초기 설정, 백업/설정을 제공한다.
- 발급한 임시 접속 코드 원문은 로컬 런처에서 다시 확인할 수 있지만 LAN의 직원용 API에는 노출하지 않는다.
- 런처 React 앱은 검증된 pywebview Python bridge로만 호스트 제어 명령을 수행하며 해당 명령을 LAN에 공개하지 않는다.
- Python과 Node.js 설치 없이 포터블 패키지로 사용하고, WebView2 Runtime 유무를 시작 시 검사해 누락 시 명확한 설치 경로를 제공한다.

## 비기능 요구사항

- 사무용 노트북에서 5명 내외 동시 사용 시 회원 검색과 신청 저장이 체감상 즉시 처리된다.
- 서버와 DB 오류는 사용자 안내와 개발자 로그를 구분한다.
- SQLite 트랜잭션, WAL, foreign key, busy timeout으로 동시 쓰기 충돌을 관리한다.
- 인증된 사용자만 업무 화면과 WebSocket 이벤트에 접근할 수 있다.
- 내부망이라도 인증, HTTPS, CSRF 방어, 로그인 실패 제한을 프로토타입 완료 조건으로 둔다.
- 전화번호, 생년월일, 백업 파일이 로그나 저장소에 불필요하게 노출되지 않는다.

## 보류 범위
