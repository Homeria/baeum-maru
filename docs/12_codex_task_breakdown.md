# 배움마루 Codex 작업 단위

## 1. 목적

이 문서는 Codex와 함께 배움마루를 만들 때 작업을 작게 나누기 위한 기준이다.

각 작업은 다음을 가져야 한다.

- 목표
- 변경 범위
- 완료 기준
- 확인 방법

## 2. 작업 원칙

- 한 번에 하나의 개념을 구현한다.
- 문서, 코드, 테스트를 같은 작업에서 필요한 만큼만 수정한다.
- 새 기능은 가능한 한 수동 확인 경로를 만든다.
- DB schema 변경은 migration과 함께 한다.
- 구현 후 다음 작업을 더 쉽게 만들도록 작은 README 또는 주석을 남긴다.

## 3. MVP-0 작업

### Task 01 Go 프로젝트 뼈대

목표:

- Go module 생성
- 기본 디렉터리 생성
- 서버 진입점 생성

완료 기준:

```text
go test ./...
```

명령이 성공한다.

### Task 02 설정 로더

목표:

- `config.json` 읽기
- 없으면 기본값 생성
- 서버 포트, DB 경로, 백업 경로 설정

완료 기준:

- 기본 config 생성
- 잘못된 config 오류 메시지 확인

### Task 03 로깅

목표:

- `logs/app.log` 생성
- `slog` 기반 로거 구성
- 사용자 메시지와 개발자 로그 분리 기준 마련

완료 기준:

- 앱 실행 로그가 파일에 기록됨

### Task 04 HTTP 서버

목표:

- `cmd/server`에서 HTTP 서버 실행
- `/`, `/admin`, `/reception` 기본 페이지 제공
- graceful shutdown 구조 추가

완료 기준:

- 브라우저에서 세 경로 접속 가능

### Task 05 SQLite 연결

목표:

- `data/center.db` 자동 생성
- WAL, foreign key, busy timeout 설정
- DB 연결 헬퍼 구현

완료 기준:

- 서버 실행 시 DB 파일 생성
- PRAGMA 설정 확인

### Task 06 마이그레이션 러너

목표:

- `schema_migrations` 테이블 생성
- SQL migration 실행
- 이미 실행한 migration 재실행 방지

완료 기준:

- 빈 DB에서 테이블 생성
- 재실행해도 오류 없음

### Task 07 회원 CRUD

목표:

- 회원 등록
- 회원 검색
- 회원 수정
- 회원 상세 조회

완료 기준:

- 웹 화면에서 회원을 등록하고 검색할 수 있음

### Task 08 강좌 CRUD

목표:

- 회차 생성
- 강좌 분류 생성
- 강좌 생성
- 강좌 개설 생성
- 시간표 등록

완료 기준:

- 웹 화면에서 강좌 목록을 볼 수 있음

### Task 09 신청 등록

목표:

- 회원 선택
- 강좌 선택
- 신청 저장
- 강좌별 신청자 목록 조회

완료 기준:

- 한 회원을 한 강좌에 신청할 수 있음

## 4. MVP-1 작업

### Task 10 접수 화면 개선

목표:

- 회원 검색 부분 갱신
- 선택 회원 신청 목록 표시
- 신청 저장 후 목록 갱신

완료 기준:

- 접수 직원 흐름으로 연속 신청 가능

### Task 11 신청 제한 규칙

목표:

- 같은 강좌 중복 신청 금지
- 같은 시간대 중복 신청 금지
- 최대 신청 수 제한
- 접수 마감 후 수정 금지

완료 기준:

- 각 규칙 위반 시 사용자 메시지 표시

### Task 12 신청 상태 이력

목표:

- 신청 상태 변경 시 history 기록
- 취소와 재신청 정책 구현

완료 기준:

- 신청 취소 이력이 남음

## 5. MVP-2 작업

### Task 13 추첨 준비 화면

목표:

- 강좌별 신청 수 표시
- 정원 초과 여부 표시
- 추첨 대상 선택

완료 기준:

- 추첨 대상 강좌를 확인할 수 있음

### Task 14 추첨 실행

목표:

- seed 기반 추첨
- 당첨자 저장
- 대기자 순번 생성
- 신청 상태 갱신

완료 기준:

- 같은 seed와 같은 입력으로 같은 결과 생성

### Task 15 엑셀 출력

목표:

- 강좌별 신청자 명단 출력
- 추첨 결과 출력
- 대기자 명단 출력
- 출석부 출력

완료 기준:

- `exports/`에 xlsx 파일 생성

## 6. MVP-3 작업

### Task 16 백업

목표:

- 수동 백업
- 최근 백업 시간 조회
- 백업 후 종료

완료 기준:

- `backups/`에 DB 백업 파일 생성

### Task 17 복구

목표:

- 복구 전 현재 DB 자동 백업
- 백업 파일에서 복구
- 복구 후 상태 확인

완료 기준:

- 백업 파일로 이전 상태를 복구할 수 있음

### Task 18 콘솔형 포터블 런처

목표:

- `cmd/launcher`에서 앱 런타임 조립
- HTTP 서버 시작
- 기본 브라우저 열기
- 종료 신호 수신 시 graceful shutdown
- 패키징된 exe에서 같은 흐름 수행

완료 기준:

- exe 실행 후 브라우저에서 관리자/접수 화면 접속 가능
- 종료 시 서버가 정상 종료됨

비고:

- Fyne 기반 네이티브 런처는 후속 사용성 개선 후보로 둔다.

### Task 19 포터블 패키징

목표:

- Windows exe 빌드
- portable ZIP 생성
- 처음 사용 README 포함

완료 기준:

- 압축 해제 후 실행 가능

## 7. 작업 요청 예시

Codex에 요청할 때는 다음처럼 말한다.

```text
docs 기준으로 MVP-0 Task 04 HTTP 서버를 구현해줘.
구현 후 go test ./...까지 확인해줘.
```

```text
Task 11 신청 제한 규칙 중 동일 시간대 중복 신청 금지만 먼저 구현해줘.
테스트 케이스도 같이 추가해줘.
```

## 8. 커밋 메시지 예시

```text
docs: 현재 구현 상태 문서화
- MVP 진행 상태 정리
- 다음 브랜치 계획 추가
```

```text
feat: 설정 로더 추가
- 기본 config 생성 지원
- 설정 검증 테스트 추가
```

```text
feat: 회원 신청 흐름 추가
- 회원 검색 후 강좌 신청 지원
- 신청 목록 갱신 처리
```

```text
test: 신청 제한 규칙 테스트 추가
- 중복 신청 차단 검증
- 시간대 충돌 차단 검증
```

## 10. 공개 저장소 정리 작업

### Task 20 비상업 소스 공개 준비

목표:

- README 실행/패키징 안내 정리
- PolyForm Noncommercial License 1.0.0 적용
- NOTICE와 Required Notice 추가
- `.gitignore` 산출물 제외 규칙 점검
- PR/Issue 템플릿 추가
- 릴리즈 전 체크리스트 추가

완료 기준:

- 공개 저장소에 실데이터가 포함되지 않는다.
- 새 기여자가 실행과 검증 방법을 README에서 찾을 수 있다.
- 릴리즈 전 확인할 항목이 문서화되어 있다.
