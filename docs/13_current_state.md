# 배움마루 현재 구현 상태

## 1. 목적

이 문서는 문서상 계획과 실제 구현 상태를 맞추기 위한 기준 문서이다.

배움마루는 현재 MVP-0부터 MVP-3까지의 핵심 업무 흐름을 대부분 구현한 상태이며, 이후 작업은 기능 확장보다 사용성, 가져오기, 운영 안정성, 배포 준비를 우선한다.

## 2. 구현 완료로 보는 범위

### 기반 구조

- Go module과 `cmd/server`, `cmd/launcher` 진입점
- 설정 파일 로더와 기본값 생성
- 파일 로깅
- SQLite 연결, WAL, foreign key, busy timeout
- SQL 마이그레이션 실행
- GitHub Actions 기반 Go 빌드/테스트 CI

### 업무 기능

- 회원 등록/검색
- 강좌, 회차, 강좌 개설, 시간표 기반 관리
- 수강신청 등록과 취소
- 회원별/강좌별 신청 조회
- 동일 강좌 중복 신청 차단
- 동일 시간대 중복 신청 차단
- 1인 최대 신청 수 제한
- 접수 마감 후 변경 차단
- 신청 상태 이력 기록

### 추첨과 상태 관리

- 강좌 개설별 추첨 실행
- 정원 이하 신청 자동 선발
- 정원 초과 신청 seed 기반 무작위 선발
- 대기자 순번 생성
- 추첨 결과 저장
- 신청 상태 갱신
- 선발자 확정
- 취소 후 대기자 승격

### 엑셀 출력

- 회원 목록 내보내기
- 강좌 개설 목록 내보내기
- 신청 목록 내보내기
- 추첨 결과 내보내기
- 출석 세션별 출석부 내보내기
- 강좌 개설별 출석부 내보내기

### 백업/복구

- 수동 백업 파일 생성
- 백업 목록 조회
- 백업 파일 다운로드
- 복구 요청 예약
- 다음 실행 시 복구 적용
- 복구 전 현재 DB 자동 백업

### 출석

- 출석 세션 생성
- 확정 신청자 기준 출석 대상 조회
- 출석 상태 저장
- 출석 엑셀 출력

### 포터블 패키징

- Windows 실행 파일 빌드
- `config.json`, 런타임 폴더, 처음 사용 안내 포함
- portable ZIP 생성
- 실행 시 서버 시작과 브라우저 열기

## 3. 의도적으로 남겨둔 공백

## 5. MVP 완료 판단

현재 구현은 업무 흐름만 보면 MVP-3에 가깝다. 다만 실사용 관점에서는 다음을 더 보강한 뒤 MVP 완료로 본다.

- 담당자가 길게 설명을 듣지 않아도 화면을 사용할 수 있다.
- 표준 엑셀 양식으로 회원과 강좌를 가져올 수 있다.
- 추첨 재실행 시 기존 결과 처리 정책이 명확하다.
- 백업 상태와 복구 절차가 화면에서 충분히 이해된다.
- 더미 데이터 기준 전체 업무 시뮬레이션이 통과한다.
