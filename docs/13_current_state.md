# 배움마루 현재 구현 상태

## 1. 목적

이 문서는 문서상 계획과 실제 구현 상태를 맞추기 위한 기준 문서이다.

배움마루는 현재 MVP-0부터 MVP-3까지의 핵심 업무 흐름을 대부분 구현한 상태이며, 이후 작업은 기능 확장보다 사용성, 가져오기, 운영 안정성, 배포 준비를 우선한다.

## 2. 구현 완료로 보는 범위

### 기반 구조

- Go module과 `cmd/server`, `cmd/launcher` 진입점
- 설정 파일 로더와 기본값 생성
- 파일 로깅
- SQLite 연결, WAL, foreign key, busy timeout
- SQL 마이그레이션 실행
- GitHub Actions 기반 Go 빌드/테스트 CI
- GitHub Actions 기반 Windows 포터블 패키징 workflow
- 접속 코드 인증 모델과 config 비밀번호 fallback
- 서명된 세션 쿠키 기반 접근 보호
- 런처/직원/임시 직원/조회 전용 역할별 접근 권한
- Fyne 빌드 태그 기반 런처 UI, 서버 상태, 접속 코드 발급/관리, 작업 로그 패널
- 공개 저장소용 README, LICENSE, NOTICE, 기여/보안 안내, PR/Issue 템플릿

### 업무 기능

- 회원 등록/검색/필터/정렬/수정
- 강좌, 회차, 강좌 개설, 시간표 기반 등록/필터/정렬/수정
- 수강신청 등록과 취소
- 회원별/강좌별 신청 조회와 신청 현황 필터/정렬
- 동일 강좌 중복 신청 차단
- 동일 시간대 중복 신청 차단
- 1인 최대 신청 수 제한
- 접수 마감 후 변경 차단
- 신청 상태 이력 기록
- 엑셀 표준 양식 기반 회원/강좌 가져오기
- 운영 설정 화면
- 감사 로그 기록과 조회

### 추첨과 상태 관리

- 강좌 개설별 추첨 실행
- 정원 이하 신청 자동 선발
- 정원 초과 신청 seed 기반 무작위 선발
- 대기자 순번 생성
- 추첨 결과 저장
- 신청 상태 갱신
- 선발자 확정
- 취소 후 대기자 승격

### 엑셀 출력

- 회원 목록 내보내기
- 강좌 개설 목록 내보내기
- 신청 목록 내보내기
- 추첨 결과 내보내기
- 출석 세션별 출석부 내보내기
- 강좌 개설별 출석부 내보내기

### 백업/복구

- 수동 백업 파일 생성
- 백업 목록 조회
- 백업 파일 다운로드
- 복구 요청 예약
- 다음 실행 시 복구 적용
- 복구 전 현재 DB 자동 백업

### 출석

- 출석 세션 생성
- 확정 신청자 기준 출석 대상 조회
- 출석 상태 저장
- 강좌 선택 목록 정렬
- 출석 엑셀 출력

### 포터블 패키징

- Windows 실행 파일 빌드
- `config.json`, 런타임 폴더, 처음 사용 안내 포함
- portable ZIP 생성
- 실행 시 서버 시작과 브라우저 열기
- GitHub Actions artifact 기반 portable ZIP 생성

### 검증

- 실제 SQLite 기반 MVP 업무 흐름 시뮬레이션 테스트
- 회원, 강좌, 신청, 추첨, 출석, 엑셀 출력, 백업, 감사 로그 통합 검증

## 5. MVP 완료 판단

현재 구현은 업무 흐름 기준 MVP-3 핵심 범위를 충족한다. 다만 실사용 관점에서는 다음을 더 보강한 뒤 첫 공개 릴리즈로 본다.

- Windows 실기기에서 포터블 ZIP 실행이 확인된다.
- 담당자가 짧은 사용자 가이드만 보고 핵심 업무를 수행할 수 있다.
- 실사용 규모에 가까운 더미 데이터로 주요 화면과 엑셀 출력이 확인된다.
- 공개 저장소에 실데이터와 개인정보가 포함되지 않는다.
