# 화면 흐름

## 화면 분리 원칙

- 런처는 호스트의 서버/운영 콘솔이다.
- 웹은 접수 직원과 관리자가 반복적으로 쓰는 업무 화면이다.
- 같은 데이터를 여러 화면에서 보여주더라도, 각 화면의 주된 작업을 하나로 유지한다.

## Wails 런처

```text
대시보드
├─ 서버 시작/중지/재시작
├─ 내부망 접속 주소와 포트
├─ DB/백업/최근 오류 요약
접속 관리
├─ 접속 코드 발급
├─ 사용 가능/만료/폐기 목록
└─ 상세, 연장, 폐기, 삭제
공간 설정
├─ 건물/층
├─ 공간
└─ 역할
로그
설정
백업/실험실
```

탭은 화면을 바꿔도 상태를 유지한다. 런처는 시작 시 서버를 자동으로 열거나 브라우저를 강제로 띄우지 않는다.

## 웹 앱

```text
로그인
├─ 접수
│  ├─ 회원 검색 또는 신규 회원 입력
│  ├─ 선택한 회원의 기존 신청 요약
│  ├─ 희망 강좌 다중 선택
│  └─ 한 번의 검증/저장
├─ 회원
│  ├─ 검색/필터/목록
│  ├─ 상세와 수정
│  └─ 신청 이력
├─ 강좌 운영
│  ├─ 회차/분류/과목/강사/공간 기준 데이터
│  └─ 강좌 개설과 복수 시간표
├─ 신청 현황
├─ 추첨
├─ 출석/출력
└─ 설정/감사 로그
```

## 접수 흐름

1. 직원이 회원번호, 이름, 연락처 일부로 검색한다.
2. 기존 회원을 선택하거나 새 회원 정보를 입력한다.
3. 강좌를 최대 신청 가능 수만큼 고른다. 선택 즉시 시간 충돌, 중복, 접수 상태, 정원 정보를 보여준다.
4. 저장 시 서버가 같은 규칙을 트랜잭션으로 다시 검증한다.
5. 성공하면 회원 정보와 여러 신청을 함께 반영하고, 다른 화면의 관련 목록은 SSE 이벤트로 갱신한다.

## 충돌과 갱신 UX

- 목록 화면은 관련 이벤트가 오면 query를 다시 읽는다.
- 폼 편집 중에는 자동 리로드하지 않고 새 데이터 알림을 보인다.
- 저장 충돌은 "다른 직원의 작업으로 조건이 바뀌었습니다"와 함께 최신 데이터를 다시 보여준다.
- 추첨/복원처럼 큰 변경을 수행할 때는 진행 상태와 편집 잠금 이유를 명확히 표시한다.
