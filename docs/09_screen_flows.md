# 배움마루 화면 흐름

## 1. 원칙

- 담당자는 명령어를 입력하지 않는다.
- 접수 직원은 빠르게 검색하고 신청을 저장할 수 있어야 한다.
- 화면은 설명보다 상태와 다음 행동을 분명히 보여준다.
- 관리자 기능과 접수 기능을 분리한다.
- 출력 화면은 인쇄와 엑셀 저장을 우선한다.

## 2. 전체 화면 지도

```text
Fyne 런처
├─ 서버 시작/중지
├─ 관리 화면 열기 -> /admin
├─ 접수 화면 열기 -> /reception
├─ 백업하기
└─ 백업 후 종료

웹
├─ /admin
│  ├─ 회원 관리
│  ├─ 강좌 관리
│  ├─ 신청 현황
│  ├─ 추첨 관리
│  ├─ 출력
│  └─ 설정
├─ /reception
│  ├─ 회원 검색
│  ├─ 신청 입력
│  └─ 신청 변경/취소
└─ /reports
   ├─ 강좌별 명단
   ├─ 추첨 결과
   ├─ 대기자 명단
   └─ 출석부
```

## 3. 런처 컨트롤 패널

### 3.1 표시 정보

- 프로그램명
- 서버 상태
- 호스트 IP
- 포트
- 접속 주소
- DB 상태
- DB 파일 위치
- 최근 백업 시간
- 접속 중인 클라이언트 수
- 최근 오류 또는 로그 요약

### 3.2 주요 행동

```text
서버 시작
→ 설정 로드
→ DB 연결 확인
→ 마이그레이션 실행
→ HTTP 서버 시작
→ 접속 주소 표시
```

```text
백업 후 종료
→ 접수 잠금
→ SQLite 안전 백업
→ 서버 graceful shutdown
→ 프로그램 종료
```

### 3.3 상태

```text
stopped
starting
running
stopping
error
```

## 4. 관리자 대시보드

### 목적

담당 관리자가 현재 접수 상태를 한눈에 확인한다.

### 표시 정보

- 현재 회차
- 접수 상태
- 회원 수
- 강좌 수
- 총 신청 수
- 정원 초과 강좌 수
- 최근 신청
- 최근 백업 시간

### 주요 행동

- 회원 관리로 이동
- 강좌 관리로 이동
- 신청 현황으로 이동
- 추첨 관리로 이동
- 출력 화면으로 이동

## 5. 회원 관리

### 목록/검색

입력:

- 이름
- 회원번호
- 연락처 일부

결과:

- 회원번호
- 이름
- 성별
- 생년월일
- 연락처
- 최근 신청 수

행동:

- 회원 상세 보기
- 회원 수정
- 새 회원 등록
- 엑셀 가져오기
- 엑셀 내보내기

### 회원 등록/수정

필수:

- 이름

선택:

- 회원번호
- 성별
- 생년월일
- 연락처
- 비고

검증:

- 회원번호 중복 경고
- 이름과 생년월일이 같은 회원이 있으면 유사 중복 경고

## 6. 강좌 관리

### 강좌 목록

표시:

- 강좌명
- 분류
- 강사
- 요일/시간
- 정원
- 신청 수
- 접수 가능 여부

행동:

- 강좌 등록
- 강좌 수정
- 강좌 비활성화
- 신청자 목록 보기

### 강좌 등록/수정

필수:

- 강좌명
- 회차
- 정원
- 요일/시간

선택:

- 분류
- 강사
- 강의실
- 비고

검증:

- 종료 시간이 시작 시간보다 늦어야 한다.
- 정원은 0 이상이어야 한다.
- 같은 회차의 같은 강좌 중복 개설은 기본적으로 막는다.

## 7. 접수 화면

### 핵심 흐름

```text
회원 검색
→ 회원 선택
→ 현재 신청 목록 확인
→ 신청 가능한 강좌 검색
→ 강좌 선택
→ 제한 규칙 검사
→ 신청 저장
→ 신청 목록 갱신
```

### 화면 구성

- 상단: 회원 검색
- 왼쪽: 선택된 회원 정보
- 가운데: 신청 가능한 강좌 목록
- 오른쪽: 선택 회원의 신청 목록
- 하단: 사용자 안내 메시지

### 빠른 입력 기준

- 검색 결과는 부분 갱신한다.
- 신청 저장 후 검색어를 유지한다.
- 같은 회원에게 여러 강좌를 연속 신청할 수 있게 한다.
- 오류 메시지는 어떤 규칙 때문에 막혔는지 보여준다.

## 8. 신청 현황

### 관리자용 조회

필터:

- 회차
- 강좌
- 상태
- 회원명

표시:

- 강좌명
- 회원명
- 신청 상태
- 신청 시간
- 수정 시간

행동:

- 신청 취소
- 상태 변경
- 강좌별 명단 출력

## 9. 추첨 관리

### 추첨 준비 화면

표시:

- 강좌별 정원
- 신청 수
- 초과 여부
- 추첨 필요 여부
- 마지막 추첨 실행 여부

행동:

- 추첨 대상 선택
- seed 자동 생성
- seed 직접 입력
- 추첨 전 백업
- 추첨 실행

### 추첨 결과 화면

표시:

- 당첨자
- 대기자
- 탈락자
- seed
- 실행 시각

행동:

- 결과 엑셀 저장
- 대기자 명단 저장
- 출석부 생성

## 10. 출력 화면

출력 종류:

- 회원 명단
- 강좌별 신청자 명단
- 당첨자 명단
- 대기자 명단
- 출석부

출력 흐름:

```text
출력 종류 선택
→ 회차 선택
→ 강좌 선택 또는 전체 선택
→ 미리보기
→ 엑셀 저장
```

## 11. 설정 화면

초기 설정:

- 표시명
- 포트
- DB 경로
- 백업 경로
- 기본 회차
- 1인 최대 신청 강좌 수

주의:

- DB 경로 변경은 서버 중지 상태에서만 허용한다.
- 포트 변경은 서버 재시작 후 적용한다.

## 12. 오류 메시지 기준

좋은 메시지:

```text
이미 신청한 강좌입니다.
```

```text
이 회원은 같은 시간대의 다른 강좌를 이미 신청했습니다.
```

```text
접수가 마감되어 신청을 수정할 수 없습니다.
```

피해야 할 메시지:

```text
constraint failed
```

```text
500 internal server error
```
